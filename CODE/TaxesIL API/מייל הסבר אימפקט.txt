התהליך מורכב למספר חלקים:
1. רישום העסק כבית תוכנה ברשות המיסים באם אתה עדיין לא רשום כך (באחריות הלקוח) 
2. רישום כפול (צד לקוח וצד מתכנת) לשירותי ה API (הרישום הינו לכלל השירותים ולאחר מכן ניתן לבחור את המודולים הרצויים)
3. לאחר הרישום, מתן הרשאה לחשבון המתכנת 
4. יש לבצע מספר הגדרות דרך הפורטל (לאחר הרישום) ע"מ לקבל TOKEN 
5. מימוש אינטגרציה - הוספת מחלקת קוד ל"דיבור" עם רשות המיסים והגדרת המתודות הרצויות (במקרה שלנו - מודל החשבוניות) 
6. קבלת מספר הקצאה בעבור כל דיווח מקוון דרך ה API ורישום הערך ב DB של הלקוח (יש להוסיף עמודה בטבלת המסמכים ושאילתות הדרושות לעדכון)

בעיות ואתגרים: 
7. הבעיה העיקרית היא שפת הממשק, הדבר הראשון שצריך לבדוק זה האם ניתן להקים את הקוד כפלאגין חיצוני (בעזרת DLL) אחרת צריך לכתוב קוד ב ASP או VB 
8. בירור סוג התקשורת (מול הלקוח) ובדיקת תמיכה מול התיעוד של רשות המיסים (האם OAuth 2.0 או App-Level) - דבר איתי על זה וארחיב
